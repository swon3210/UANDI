# Play Console 데이터 보안(Data safety) 입력 매핑

Play Console → **앱 콘텐츠 → 데이터 보안** 폼에 그대로 옮겨 적기 위한 매핑표.
실제 동작(코드) 기준으로 작성했다. 기능이 바뀌면 이 문서와 개인정보처리방침
(`apps/web/src/app/privacy/page.tsx`)을 함께 갱신한다.

> ⚠️ **3자 일치 원칙**: 이 폼 ↔ 개인정보처리방침 ↔ 실제 앱 동작이 모두 일치해야 한다.
> Google이 가장 자주 반려하는 사유가 "선언과 실제 동작 불일치"다.

---

## 0. 핵심 판단 (먼저 읽기)

### "수집(Collected)" vs "공유(Shared)"

- **수집** = 데이터가 기기 밖으로 전송됨 → 이 앱은 거의 모든 항목이 **수집됨**.
- **공유** = **제3자(별도 회사)에게 전송**. 단, Google 정의상 아래는 "공유"에서 **제외**된다:
  - 개발자 지시에 따라 처리하는 **서비스 제공자(service provider)** 로의 전송
  - 이용자가 직접 시작한 전송 / 법적 목적 전송 / 익명화 데이터
- 따라서 **OpenAI, Firebase(Google Cloud), Vercel 은 모두 서비스 제공자**이므로
  → **"공유(Shared)" = 아니오** 로 답한다. (단, 아래 전제 충족 필요)
  - OpenAI: API 호출 데이터가 **모델 학습에 사용되지 않음**(OpenAI API 기본 정책). ✅ 충족
  - 커플 상대방과의 데이터 공유는 **앱 내 이용자 간 공유**라 "제3자 공유" 아님.
- 결론: **대부분 항목은 "수집=예 / 공유=아니오"**.

### 전역 보안 질문

| 질문                                     | 답                                                                                             |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------- |
| 전송 중 데이터 암호화(in transit)?       | **예** (HTTPS/TLS)                                                                             |
| 이용자가 데이터 삭제를 요청할 수 있는가? | **예** — 인앱 `설정 → 회원탈퇴`로 계정·데이터 삭제. 점검 첨부 이미지는 점검 완료 시 자동 삭제. |
| Play Families(아동) 정책 대상?           | **아니오** (만 14세 미만 가입 불가)                                                            |

## 1. 수집하는 데이터 유형 (폼 카테고리별)

각 항목: **수집=예**, **공유=아니오**(위 0번 사유), 필수/선택, 처리 목적.

### Personal info (개인정보)

| 데이터                         | 수집 | 출처             | 필수/선택 | 목적               |
| ------------------------------ | ---- | ---------------- | --------- | ------------------ |
| Name (이름)                    | 예   | Google 로그인    | 필수      | 계정 관리, 앱 기능 |
| Email address (이메일)         | 예   | Google 로그인    | 필수      | 계정 관리, 앱 기능 |
| User IDs (uid, 커플 식별자 등) | 예   | 로그인/커플 연결 | 필수      | 계정 관리, 앱 기능 |

> 프로필 사진 URL은 Google 계정에서 받는 값. Personal info > "기타 정보"로 묶어도 되고,
> 별도 선언이 부담되면 위 3개로 충분히 커버된다.

### Financial info (금융정보) ← 점검 기능 때문에 반드시 선언

| 데이터                                   | 수집 | 출처                                                                             | 필수/선택 | 목적    |
| ---------------------------------------- | ---- | -------------------------------------------------------------------------------- | --------- | ------- |
| **Other financial info (기타 금융정보)** | 예   | ① 가계부 입력(금액·카테고리·메모·거래일) ② **점검용 카드/계좌 거래 명세 이미지** | 선택      | 앱 기능 |

> 거래 명세 이미지의 **내용**(가맹점·금액·거래일 등)이 금융정보이므로 "기타 금융정보"로 수집 선언.
> "User payment info(결제수단)" / "Purchase history(이 앱에서의 구매 내역)" 는 **아니오** —
> 앱이 결제수단을 저장하거나 인앱 결제 내역을 만드는 게 아니다.

### Photos and videos (사진/동영상) ← 갤러리 + 명세 이미지

| 데이터            | 수집 | 출처                                                    | 필수/선택 | 목적    |
| ----------------- | ---- | ------------------------------------------------------- | --------- | ------- |
| **Photos (사진)** | 예   | ① 갤러리 업로드 사진 ② **점검용 거래 명세 캡처 이미지** | 선택      | 앱 기능 |

### App activity (앱 활동)

| 데이터                           | 수집 | 출처                                                    | 필수/선택 | 목적    |
| -------------------------------- | ---- | ------------------------------------------------------- | --------- | ------- |
| **Other user-generated content** | 예   | 캡션·태그·폴더명·가계부 메모·커뮤니티 글·AI 분석 텍스트 | 선택      | 앱 기능 |

### Device or other IDs (기기 식별자)

| 데이터                  | 수집 | 출처          | 필수/선택 | 목적                                    |
| ----------------------- | ---- | ------------- | --------- | --------------------------------------- |
| **Device or other IDs** | 예   | FCM 푸시 토큰 | 선택      | 앱 기능, 개발자 커뮤니케이션(푸시 알림) |

### App info and performance (앱 정보·성능) → 선언 안 함

| 데이터                   | 수집       | 비고                                                                                                                                                                                                                                                                                             |
| ------------------------ | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Crash logs / Diagnostics | **아니오** | 코드 확인 결과 Crashlytics·Firebase Analytics·Sentry 등 **분석/크래시 SDK 미통합**. 개인정보처리방침의 "오류 로그·접속 로그"는 Vercel/Firebase 플랫폼 운영 로그(서비스 제공자 레벨)라 별도 선언 대상 아님. 추후 Crashlytics/Analytics를 붙이면 이 항목을 "예"로 바꾸고 폼·방침을 함께 갱신할 것. |

---

## 2. 선언하지 않는(수집 안 함) 카테고리

아래는 **모두 "아니오"**: Location(위치), Health and fitness, Messages(앱 내 메시지/DM 없음),
Audio, Files and docs(이미지는 Photos로 선언), Calendar, Contacts, Web browsing history,
App activity > 설치된 앱/검색 기록.

---

## 3. 데이터 삭제 방법 (폼의 "삭제 요청" 질문)

- **계정·전체 데이터 삭제**: 앱 내 `설정 → 회원탈퇴` (`useDeleteAccount` → `deleteAccount`).
- **점검 첨부 이미지**: 점검 완료 시 Storage에서 자동 삭제(`useCompleteSettlement`).
  완료 전에도 갤러리에서 개별 삭제 가능.
- 폼에서 삭제 요청 방법을 묻거나 URL을 요구하면 위 인앱 경로를 안내. (웹 삭제 페이지가
  필요하면 별도 라우트 추가 검토 — 현재는 인앱 삭제로 요건 충족.)

## 4. 제출 전 최종 체크

- [ ] 개인정보처리방침 URL 입력: `https://uandi-web.vercel.app/privacy`
- [ ] Financial info(기타 금융정보) **수집=예** 선언했는가 (점검 기능)
- [ ] Photos **수집=예** 선언했는가 (갤러리 + 명세 이미지)
- [x] Crash logs/Diagnostics → 분석 SDK 미통합으로 "아니오" 확정
- [ ] 위 선언이 개인정보처리방침 1·4·6조 내용과 일치하는가

---

## 5. Play Console 클릭 가이드 (화면 순서대로)

경로: **Play Console → 앱 콘텐츠(App content) → 데이터 보안(Data safety) → 관리/시작**

> 콘솔 버전에 따라 질문 순서가 약간 다를 수 있으나, 묻는 내용과 답은 동일하다.

### 단계 1. 개요(Overview)

읽고 **다음**.

### 단계 2. 데이터 수집 및 보안 (전역 질문)

| 질문                                              | 답     |
| ------------------------------------------------- | ------ |
| 앱에서 필수 사용자 데이터 유형을 수집/공유하나요? | **예** |
| 수집하는 모든 데이터가 전송 중 암호화되나요?      | **예** |
| 사용자가 데이터 삭제를 요청할 방법을 제공하나요?  | **예** |

> ⚠️ 삭제 질문: 인앱 회원탈퇴(`설정 → 회원탈퇴`)가 있으니 "예". URL 입력란이 나오면
> 비워도 진행되지만, **Google은 별도 "계정 삭제" 요건에서 웹 삭제 URL을 권장**한다.
> 지금은 인앱 삭제만 있으므로, 추후 웹 삭제 페이지 추가를 검토.

### 단계 3. 데이터 유형 선택 (체크박스 — 아래만 체크, 나머지 전부 해제)

- **Personal info**: ✅ Name · ✅ Email address · ✅ User IDs
- **Financial info**: ✅ Other financial info
- **Photos and videos**: ✅ Photos
- **App activity**: ✅ Other user-generated content
- **Device or other IDs**: ✅ Device or other IDs
- 나머지 카테고리(Location, Health and fitness, Messages, Audio, Files and docs,
  Calendar, Contacts, Web browsing, App info and performance) → **체크 안 함**

### 단계 4. 각 데이터 유형 상세 (선택한 항목마다 동일 패턴)

**모든 항목 공통**: 수집(Collected) = **예** / 공유(Shared) = **아니오** /
일시적 처리(ephemeral) = **아니오**(저장하므로).

| 데이터 유형                  | 필수/선택 | 수집 목적           |
| ---------------------------- | --------- | ------------------- |
| Name                         | 필수      | 앱 기능, 계정 관리  |
| Email address                | 필수      | 앱 기능, 계정 관리  |
| User IDs                     | 필수      | 앱 기능, 계정 관리  |
| Other financial info         | 선택      | 앱 기능             |
| Photos                       | 선택      | 앱 기능             |
| Other user-generated content | 선택      | 앱 기능             |
| Device or other IDs          | 선택      | 앱 기능 (푸시 알림) |

> "필수"는 로그인이 앱 사용에 반드시 필요하기 때문(Name/Email/User IDs).
> 나머지는 해당 기능을 안 써도 앱 사용이 가능하므로 "선택".

### 단계 5. 미리보기 → 저장 → 제출

요약을 검토하고 제출. Data safety 변경은 반영에 시간이 걸릴 수 있으니,
**반영 확인 후** 프로덕션 심사를 제출한다(방침↔폼↔동작 일치 유지).
