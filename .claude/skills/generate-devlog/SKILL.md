## 역할

현재 Claude Code 대화에서 수행한 작업을 분석하고,
개발 블로그 포스트를 `content/posts/` 디렉토리에 마크다운 파일로 생성한다.

## 입력

`$ARGUMENTS` — (선택) 글의 주제나 초점을 지정할 수 있다.
지정하지 않으면 현재 대화 전체를 기반으로 자동 판단한다.

## 실행 순서

### 1단계: 설정 파일 읽기

**반드시 `content/blog-config.md`를 먼저 읽는다.**

이 파일에 다음이 정의되어 있다:

- 대상 독자
- 톤 & 어조
- 글 길이 (최소/최대)
- 필수 섹션과 선택 섹션
- 코드 블록 규칙
- 태그 규칙

설정 파일의 내용을 글 생성의 최우선 제약으로 따른다.
설정 파일이 없으면 아래 기본값을 사용한다:

- 어조: 자연스러운 한글
- 길이: 1000~2000자
- 필수 섹션: 배경, 구현 내용, 배운 것
- 코드 블록: 3개 이하, 20줄 이하

### 2단계: 대화 분석

현재 대화에서 다음을 파악한다:

- **무엇을 구현했는가**: 변경된 파일, 추가된 기능, 해결한 문제
- **왜 그렇게 했는가**: 기술적 결정과 그 이유, 대안 비교
- **어떤 어려움이 있었는가**: 만난 버그, 삽질, 예상과 달랐던 점
- **무엇을 배웠는가**: 새로 알게 된 라이브러리, 패턴, 개념

### 3단계: git diff 확인

`git diff HEAD~1` 또는 현재 스테이징된 변경사항을 확인하여
구체적인 코드 변경 내용을 파악한다.

### 4단계: 슬러그 결정

- `$ARGUMENTS`가 있으면 이를 기반으로 영문 slug 생성
- 없으면 구현 내용에서 핵심 키워드를 추출해 slug 생성
- 형식: 소문자 영문 + 하이픈 (예: `photo-gallery-infinite-scroll`)

### 5단계: 마크다운 파일 생성

파일 경로: `content/posts/YYYY-MM-DD-<slug>.md` (오늘 날짜 사용)

**Frontmatter:**

```yaml
---
title: '<한글 제목>'
date: 'YYYY-MM-DD'
summary: '<1~2문장 한글 요약>'
tags: ['<관련 태그들>']
draft: true
---
```

**본문:** `content/blog-config.md`에 정의된 섹션 구조와 규칙을 따른다.

- 필수 섹션은 반드시 포함
- 선택 섹션은 해당 내용이 있을 때만 포함
- 코드 블록 개수/줄 수 제한을 준수
- 글 길이를 설정된 범위 내로 유지
- 설정된 톤/어조와 대상 독자에 맞게 작성

### 6단계: 결과 보고

생성된 파일 경로를 출력하고, 사용자에게 안내한다:

```
개발 일지가 생성되었습니다: content/posts/YYYY-MM-DD-<slug>.md

- `draft: true`로 생성되었습니다
- 내용을 검토/편집한 후 `draft: false`로 변경해 주세요
- `pnpm --filter blog dev`로 미리보기할 수 있습니다 (draft 글은 개발 환경에서만 표시)
```
