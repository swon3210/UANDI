# 페이지 명세: 카테고리 설정 (/cashbook/categories)

## 목적

수입·지출·Flex의 카테고리를 체계적으로 관리하고, 각 카테고리에 아이콘·색상·**하위 카테고리·설명·예시**를 부여해 입력 시 망설임을 줄이고 직관적으로 구분할 수 있게 한다.

---

## 개선 배경 (2026-05)

부부가 사용하면서 발견된 페인포인트:

1. **모호함**: "식비/디저트/데이트", "서비스 구독/정기결제/개발 인프라"처럼 경계가 흐린 카테고리가 많아 거래 입력 시 어디에 넣을지 망설인다.
2. **막연함**: "정기결제" 같은 묶음형 카테고리는 한 달/연간 합산이 얼마가 될지 추정하기 어렵다.

### 해결 방향: 2단계 계층 + 설명/예시 필드

- 카테고리를 **부모-자식 2단계**로 확장한다 (예: `식비 > 외식·장보기·디저트`).
- 각 카테고리에 **설명**(`description`)과 **예시 목록**(`examples`)을 부여해 입력 시 hint로 노출한다.
- 통계/예산은 부모 단위로 롤업, 자식 단위로도 드릴다운 가능.
- 거래의 카테고리 참조는 현행 `category: string`(이름) 유지. `categoryId` 마이그레이션 없음.

> 데이터 모델 변경과 결정 사항은 `03-domain-models.md`의 `CashbookCategory` 섹션 참고.
>
> 이 문서에 남아 있는 "재테크" 4탭 표기 등 현행 구현과의 불일치는 **별도 PR**에서 정리한다 (이번 작업 범위 밖).

## 분류 체계 보존 원칙 (계층 도입 시 주의)

자식 카테고리 도입은 **현재의 3축 분류 체계를 약화시키지 않아야** 한다.

| 축 | 모델 차원 보존 | 사용성 차원 위험 | 보존 장치 |
|---|---|---|---|
| **수입/지출/Flex** (group) | 자식이 부모 group 강제 상속 | — | 서비스 검증 |
| **정기/비정기·고정/변동** (subGroup) | 자식이 부모 subGroup 강제 상속 | 신규 부모 생성 시 어느 subGroup에 둘지 망설임은 그대로 | description 필드로 가이드 |
| **공동/각자** (subGroup의 신혼부부 축) | 자식이 부모 subGroup 강제 상속 | **자식 의미 분류(외식/장보기 등)에 몰입하면 "우리 돈/내 돈" 축이 시각적으로 흐려짐** | 자식 chip·폼 곳곳에 "공통/각자" 뱃지 노출 (아래 UI 규칙) |

### UI 차원의 보존 장치 (필수)

1. **자식 chip 위에 부모 subGroup 뱃지 상시 노출** — 카테고리 관리 페이지·EntryForm·통계 어디서든 자식 chip 옆/위에 `[공통]` `[각자]` `[고정]` 같은 작은 뱃지를 표시한다. 자식만 봐도 어느 축인지 즉각 인식되도록.
2. **자식 추가 시트의 부모 잠금 영역에 subGroup 명시** — 잠긴 부모 표시는 `🍚 식비 (공통)` 처럼 subGroup 라벨을 함께 노출한다. 자식을 만드는 그 순간 어느 축인지 한 번 더 환기.
3. **첫 사용 가이드는 부모를 명시한 문구로** — "식비 → 장보기/외식" 같은 모호한 가이드를 쓰지 않는다. "**공통-식비 → 장보기/외식**, **개인-식비 → 점심/간식**"처럼 부모를 명시해 자식 의미가 공통/각자별로 달라지도록 유도한다.

## 카테고리 체계

### 2.1 수입 카테고리

| 구분 | 카테고리 | 기본 아이콘 | 기본 색상 | 설명 |
|------|----------|-------------|-----------|------|
| **정기 수입** | 정기급여 | 💰 | `--income` | 매달 고정으로 들어오는 급여 |
| | 상여 | 🎁 | `--income` | 분기/반기/연말 상여금 |
| **비정기 수입** | 인센티브 | 🏆 | `--income-light` | 성과 기반 인센티브 |
| | 부업 | 💼 | `--income-light` | 부업·프리랜서 수입 |
| | 중고거래 | 🔄 | `--income-light` | 중고물품 판매 수익 |

### 2.2 지출 카테고리

#### 고정 지출 (매달 일정하게 나가는 비용)

| 카테고리 | 기본 아이콘 | 설명 |
|----------|-------------|------|
| 월세 | 🏠 | 월세·전세 이자 |
| 금융비용 | 🏦 | 대출 이자, 카드 연회비 등 |
| 보험 | 🛡️ | 건강보험, 자동차보험 등 |
| 공과금 | 💡 | 전기·수도·가스·인터넷 |
| 명절 용돈 | 🧧 | 명절·경조사 관련 고정 지출 |

#### 변동 지출 — 공통 (커플 공동 지출)

| 카테고리 | 기본 아이콘 | 설명 |
|----------|-------------|------|
| 식비 | 🍚 | 장보기, 외식, 배달 (공동) |
| 소모품 | 🧴 | 생활용품, 소모품 |
| 패션 | 👗 | 의류·신발·액세서리 (공동 구매) |
| 데이트 | 💕 | 데이트 비용 |
| 건강 | 💪 | 헬스장, 건강식품 등 (공동) |

#### 변동 지출 — 각자 (개인별 지출)

| 카테고리 | 기본 아이콘 | 설명 |
|----------|-------------|------|
| 식비 | 🍱 | 개인 점심, 간식 |
| 사회생활 | 🤝 | 회식, 모임 |
| 친구 | 🎉 | 친구 약속, 경조사 |
| 병원 | 🏥 | 개인 병원비 |
| 교통 | 🚌 | 출퇴근, 교통비 |
| 가족 | 👨‍👩‍👧 | 친정·시댁 용돈, 선물 |
| 자기계발 | 📚 | 도서, 강의, 자격증 |

> **공통 vs 각자 구분**: 항목 추가 시 "공통" 또는 "나의 지출"을 선택한다. 각자 지출은 해당 유저에게만 귀속되며, 상대방도 조회는 가능하다.

### 2.3 재테크 카테고리

| 구분 | 카테고리 | 기본 아이콘 | 설명 |
|------|----------|-------------|------|
| **현금 보유** | 예적금 | 🏦 | 정기예금, 적금 |
| | 입출금 | 💳 | 입출금 통장 잔고 |
| **투자** | 국내주식 | 📈 | 국내 주식 투자 |
| | 해외주식 | 🌍 | 해외 주식·ETF 투자 |
| | 채권 | 📄 | 국채·회사채 등 |

> **재테크 가능 금액** = `총 수입 - 총 지출`로 자동 계산되어 표시.

### 2.4 Flex 카테고리

| 구분 | 카테고리 | 기본 아이콘 | 설명 |
|------|----------|-------------|------|
| **공동 Flex** | 여행 | ✈️ | 여행 경비 |
| | 여가 | 🎬 | 영화, 공연, 문화생활 |
| **각자 Flex** | 소비 | 🛍️ | 각자 자유 소비 |
| | 여가 | 🎮 | 각자 취미·여가 |

> Flex는 예산 한도 내에서 자유롭게 쓸 수 있는 금액. 재테크 배분 후 남은 금액에서 설정.

## UI 구성

### 카테고리 관리 대시보드 (/cashbook/categories)

부모 카테고리를 카드 형태로 묶고, 그 아래에 자식 카테고리를 들여쓰기 chip으로 배치한다. 부모 카드에는 `+ 하위 추가` 인라인 버튼을 둔다.

```
┌─────────────────────────┐
│  카테고리 설정      [←]  │
├─────────────────────────┤
│  ┌─── 탭 ───────────┐  │
│  │  수입 │ 지출 │ Flex │
│  └──────────────────┘  │
│                         │
│  ── 변동 지출 (공통) ──  │  ← subGroup 섹션 헤더
│  ┌───────────────────┐  │
│  │ 🍚 식비          [⋮]│  │  ← 부모 카테고리
│  │   "장보기/외식/배달"│  │  ← description (1줄, 회색 텍스트)
│  │   └ 🥢 외식 [공통] │  │  ← 자식 chip + subGroup 뱃지 상시
│  │   └ 🛒 장보기[공통]│  │
│  │   └ 🍰 디저트[공통]│  │
│  │   [+ 하위 추가]    │  │  ← 인라인 버튼
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │ 🧴 소모품        [⋮]│  │  ← 부모 (자식 없음)
│  │   [+ 하위 추가]    │  │
│  └───────────────────┘  │
│                         │
│  [+ 카테고리 추가]       │
│                         │
├─────────────────────────┤
│  [홈] [사진] [가계부]   │
└─────────────────────────┘
```

**상호작용 규칙**

- 부모 카드의 `[⋮]` 메뉴: 편집 / 삭제 / **하위 카테고리 추가**.
- 자식 chip의 long-press 또는 chip 옆 `⋮` 아이콘: 자식 편집 / 삭제.
- 자식이 있는 부모를 삭제하려고 하면 다이얼로그에 "하위 카테고리 N개도 함께 삭제됩니다" 경고 표시.
- 자식 카테고리는 부모와 다른 subGroup/group으로 옮길 수 없다. 부모를 옮기면 자식도 함께 이동.

### 카테고리 추가/편집 — Bottom Sheet

부모와 자식 모두 같은 시트를 사용하되, **하위 카테고리 추가**로 진입한 경우 부모 정보가 잠긴 상태로 표시된다.

```
┌─────────────────────────┐
│  카테고리 추가            │
│                         │
│  부모 카테고리            │  ← 신규: 자식 추가 모드일 때만 표시
│  ┌───────────────────┐  │
│  │  🍚 식비 [공통] 🔒 │  │  ← 자식 모드는 read-only, subGroup 뱃지 함께
│  └───────────────────┘  │
│                         │
│  이름                   │
│  ┌───────────────────┐  │
│  │  (예: 외식)        │  │
│  └───────────────────┘  │
│                         │
│  설명 (선택)             │  ← 신규 필드
│  ┌───────────────────┐  │
│  │ 이 카테고리에 어떤 │  │
│  │ 항목이 들어가나요? │  │
│  └───────────────────┘  │
│                         │
│  예시 항목 (선택)        │  ← 신규 필드 (Tag 입력)
│  ┌───────────────────┐  │
│  │ [장보기] [외식] [+]│  │  ← 자유 입력 → Enter로 칩 추가
│  └───────────────────┘  │
│                         │
│  아이콘                 │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐  │
│  │🍚│ │🥢│ │🛒│ │🍰│  │
│  └──┘ └──┘ └──┘ └──┘  │
│                         │
│  색상                   │
│  ● ● ● ● ● ● ● ●     │
│                         │
│  구분 (subGroup)         │  ← 부모 모드만 표시, 자식 모드는 잠김
│  ┌────┐ ┌────┐ ┌────┐  │
│  │공통│ │ 각자│ │고정│  │
│  └────┘ └────┘ └────┘  │
│                         │
│  ┌───────────────────┐  │
│  │  저장              │  │
│  └───────────────────┘  │
└─────────────────────────┘
```

**필드 유효성**

- 이름: 1자 이상. 같은 부모 아래에서 유니크.
- 설명: 0~140자 (한 줄 hint로 잘리지 않는 길이).
- 예시: 0~8개, 각 항목 1~20자.
- 자식 모드에서 `parentCategoryId`는 자동 설정되어 변경 불가.

## 기본 프리셋 초기화

커플 생성 시 위 표의 카테고리를 기본값으로 자동 생성한다. 사용자는 이후 추가/편집/삭제/순서 변경이 가능하다.

- 기본 프리셋은 `isDefault: true`로 생성
- 기본 프리셋도 삭제 가능하되, 해당 카테고리를 사용 중인 항목이 있으면 확인 다이얼로그 표시
- 카테고리 삭제 시 기존 항목의 카테고리는 유지 (orphaned 허용)

## 항목 추가 시 카테고리 선택 UX

거래 입력 폼(`EntryForm` > `CategoryChips`)은 다음과 같이 동작한다.

### 1) 부모-자식 chip 표시 규칙

- subGroup 섹션 안에서 **부모 chip + 그 아래 자식 chip 묶음**을 한 그룹으로 묶어 보여준다.
- 자식이 없는 부모는 단독 chip으로 표시 (현행 동일).
- 자식이 있는 부모를 탭하면 → 자식 chip만 펼치는 인라인 expand. 부모 자체도 선택 가능하지만 "미분류" 의미로 시각적 강조 약화.
- 자식 chip 선택 시 폼 라벨에 **브레드크럼**으로 컨텍스트 표시: `식비 [공통] > 외식`.
- 자식 chip 옆에는 **부모 subGroup 뱃지를 상시 노출**한다 (`[공통]` `[각자]` `[고정]` 등). "공동/각자" 축이 자식 단계에서 시각적으로 사라지지 않도록 보존하는 핵심 장치.

```
│  ── 변동 지출 (공통) ──               │
│  ┌─────────────┐                       │
│  │ 🍚 식비 ▾    │  ← 부모 chip (자식 펼침 토글)
│  │   🥢 외식 [공통]   │
│  │   🛒 장보기 [공통] │
│  │   🍰 디저트 [공통] │
│  └─────────────┘                       │
│  [🧴 소모품] [👗 패션] [💕 데이트]    │  ← 자식 없는 부모는 단독
│  [+ 카테고리 추가]                    │
```

### 2) 설명/예시 hint 노출

선택된 카테고리 아래에 description + examples를 보조 텍스트로 표시한다.

```
│  카테고리: 🍚 식비 > 🛒 장보기         │
│  ↳ "마트·온라인 장보기. 외식·디저트   │
│      제외." 예시: 코스트코, 쿠팡, 마트  │
```

### 3) 예시(`examples`) 기반 자동완성

- 거래 메모(`description`) 입력 시 모든 카테고리의 `examples` 배열을 sub-string 매칭하여 상위 3개 카테고리를 **추천 chip**으로 띄운다.
- 예: 메모에 "코스트코" 입력 → "🛒 장보기" 카테고리 추천.
- 추천 chip을 탭하면 카테고리가 자동 선택된다. 단순 매칭만 사용(머신러닝 없음).

### 4) Orphan 처리

- 기존 거래의 `category` 이름이 어떤 카테고리에도 매칭되지 않으면 "선택됨" 섹션에 단독 chip으로 표시 (현행 동일).
- 부모만 있던 카테고리에 새로 자식이 추가된 경우, 기존 거래는 부모 이름 그대로 유지된다 (자동 재분류 안 함). 통계에서는 부모의 "미분류" 묶음으로 집계.

---

## 통계/예산에서의 계층 처리

- **월간/주간 대시보드**: 부모 카테고리 단위로 합계 표시. 부모를 탭하면 자식별 breakdown을 드릴다운으로 노출.
- **연간 예산(`AnnualPlanItem`)**: 1차 도입 시 **부모 카테고리에만 예산 설정 가능**. 자식별 예산은 후속 과제로 분리한다.
  - `AnnualPlanItem.categoryId`는 부모의 id만 허용 (자식 id는 거부).
  - 자식 거래 합계는 부모 예산의 소진율에 자동 반영된다.
- **카테고리 통계 차트(`CategoryDonutChart`)**: 기본은 부모 레벨. 우상단 토글로 "자식까지 펼치기" 옵션 제공.

## 권장 프리셋 (DEFAULT_CATEGORIES 보강안)

기존 `packages/cashbook-core/src/constants/default-categories.ts`의 프리셋에 `description` / `examples`를 채워 넣는다. 신규 가입자에 한해 자동 적용 (기존 커플은 영향 없음).

예시 (전체 목록은 구현 시 확정):

| 카테고리 | description (1줄) | examples |
|----------|-------------------|----------|
| 식비 (공통) | 부부가 함께한 장보기·외식·배달 | 마트, 외식, 배달 |
| 사회생활 | 회식, 동료/지인 모임 | 회식, 모임, 동료 |
| 공과금 | 전기·수도·가스·인터넷 | 전기, 수도, 가스, 인터넷 |
| 자기계발 | 도서·강의·자격증 | 책, 인강, 자격증 |
| 정기급여 | 매달 고정으로 들어오는 급여 | 월급 |

> 추천 자식 카테고리는 강제하지 않는다. 사용자가 필요에 따라 직접 만든다. 단, 첫 사용 시 온보딩 카드에서는 **부모를 명시한 가이드**를 제안한다 — 예: "**공통-식비 → 장보기 / 외식**, **개인-식비 → 점심 / 간식**". 이렇게 해야 자식 의미가 공통/각자별로 다르게 자라 "공동/각자" 축을 보존할 수 있다.
