# 페이지 명세: 가계부 (/inner/cashbook)

## 목적

커플의 수입과 지출을 함께 기록하고 월별로 관리하는 기능.

> **가계부 기능 문서 구성**
>
> | 파일 | 내용 |
> |------|------|
> | `04-cashbook.md` | 내역 CRUD + 메인 화면 (이 문서) |
> | `04a-cashbook-categories.md` | 카테고리 체계 + 관리 대시보드 |
> | `04b-cashbook-annual-plan.md` | 연간 예산 계획 |
> | `04c-cashbook-monthly.md` | 월간 대시보드 (목표 vs 실제) |
> | `04d-cashbook-weekly.md` | 주간 예산 & 지출 추적 |

---

## 사용자 스토리

- 커플로서, 이번 달에 얼마나 썼는지 한눈에 보고 싶다.
- 커플로서, 새로운 지출/수입 내역을 빠르게 추가하고 싶다.
- 커플로서, 과거 달의 내역도 돌아볼 수 있어야 한다.

### 내역 필터

요약 카드 위에 **필터바**가 항상 노출된다. 필터바는 (1) 인라인 월 이동 스테퍼(가장 흔한 동작인 월 단위 회고용)와 (2) `[필터]` 버튼으로 구성된다. 기간이 조회 범위(날짜)를 결정하고, 나머지 조건(타입·카테고리·키워드)은 클라이언트에서 결과를 좁힌다. 상세 필터는 모두 `[필터]` 버튼으로 여는 **바텀시트** 안에 모여 있다. **인라인 스테퍼와 시트의 '기간' 섹션은 동일한 기간 상태를 공유**한다.

```
┌──────────────────────────────┐
│ ‹ 2024년 1월 › [최신순▾][필터②]│  ← 필터바: 월 스테퍼 + 정렬 셀렉트 + 필터 버튼
│  ┌───────────────────┐  │     (범위 모드면 화살표 없이 라벨만, 탭 시 시트)
│  │ MonthlySummary    │  │  ← 항상 기간 전체 합계
│  └───────────────────┘  │
│   필터 결과 5건 | -45,000│  ← 필터 활성 시 보조 라인
└─────────────────────────┘

[필터] 버튼 → 바텀시트
┌─────────────────────────┐
│  필터                    │
│  🔍 [ 메모·카테고리 검색 ]│  ← 키워드 (description + 카테고리명)
│                         │
│  기간                    │
│ [이번 달][지난달][최근 3개월]│  ← 프리셋 (단일 선택)
│ [올해][직접 선택]        │
│  └ 직접 선택 시: [시작일][종료일] (date)
│                         │
│  타입                    │
│ [전체][지출][수입][Flex] │  ← 타입 Tabs (단일 선택)
│                         │
│  카테고리          3개 선택│  ← 접이식 체크박스 패널 (다중, OR)
│ ☑ 식비  ☑ 교통  ☐ 월세 ...│
│                         │
│  [초기화]        [적용]  │  ← 푸터 (적용 시 일괄 반영)
└─────────────────────────┘
```

#### 동작 규칙

| 동작 | 결과 |
|------|------|
| 인라인 `‹`/`›` 화살표 | (month 모드) 이전/다음 캘린더 월로 이동. 이번 달에서 `›`는 비활성(미래 불가) |
| 인라인 라벨(범위 모드) | 화살표 숨김, 라벨 탭 시 필터 시트 열림 |
| 인라인 정렬 셀렉트 | `최신순`(기본)/`오래된순`/`높은 금액순`/`낮은 금액순`. 즉시 반영(시트 밖) |
| 정렬 = 날짜순 | 날짜별 그룹(날짜 헤더) 렌더, 그룹·항목 모두 방향 적용 |
| 정렬 = 금액순 | 날짜 헤더 없는 평면 목록(금액 기준), 각 카드에 날짜 표기 |
| 기간 프리셋 `이번 달`(기본) | month 모드 커서를 현재 달로 → 1일~말일 |
| 기간 프리셋 `지난달` | month 모드 커서를 지난달로 |
| 기간 프리셋 `최근 3개월` | 이번 달 포함 직전 3개 캘린더 월 범위 |
| 기간 프리셋 `올해` | 올해 1월 1일~12월 31일 범위 |
| 기간 `직접 선택`(custom) | 시작일~종료일 범위(여러 달 across 가능). `종료 < 시작`이면 `적용` 비활성 |
| 키워드 입력 | `description`(메모) + `category`명에 부분일치(대소문자 무시) |
| 타입 탭 = `전체` | 모든 타입 표시 |
| 타입 탭 = `지출`/`수입`/`Flex` | 해당 타입만 표시 |
| 카테고리 미선택 | 카테고리 조건 미적용 |
| 카테고리 1개 이상 선택 | 선택된 카테고리 중 하나라도 일치(OR) |
| 타입 탭 전환(시트 내) | 카테고리 선택 자동 초기화(다른 타입 카테고리 혼선 방지) |
| 시트 `[적용]` | draft 상태를 페이지에 일괄 반영 후 시트 닫힘 |
| 시트 `[초기화]` | draft를 기본값으로 리셋(기간 포함). 시트는 닫지 않음 |
| 필터 결과 0건(필터 활성) | 전용 `EmptyState`("조건에 맞는 내역이 없어요") + `필터 초기화` → 기간·타입·카테고리·키워드 모두 리셋 |
| `AiSpendingAnalysis` 노출 | 기간이 단일 캘린더 월(month 모드)이고 무필터(타입·카테고리·키워드 없음)일 때만 |
| 카테고리 데이터 0건 | 카테고리 섹션 숨김 |

> 기간은 항상 값을 가지므로 "필터 활성(`isFilterActive`)" 판단에서 제외한다 — 페이지의 1차 날짜 컨트롤이지 좁히기 조건이 아니다.

#### 필터 결과 보조 라인

- 위치: `MonthlySummary` 카드 직하단(같은 시각적 그룹)
- 표시 조건: `typeFilter !== 'all'` 또는 카테고리 1개 이상 선택 또는 키워드 입력
- 내용: `필터 결과 N건` + 수입/지출 합계(0이 아닌 항목만)
- 기간 전체 합계는 항상 그대로 표시 — "전체 vs 필터" 비교가 가능하도록

#### 상태 관리

- 페이지 내 단일 `useState<CashbookFilterState>`(초기값 `createDefaultFilterState()` — 마운트 시점의 이번 달):
  - `period: PeriodSelection` — `{ mode: 'month'; year; month }` | `{ mode: 'last3Months' }` | `{ mode: 'thisYear' }` | `{ mode: 'custom'; start; end }`
  - `typeFilter: 'all' | 'expense' | 'income' | 'flex'`
  - `selectedCategoryNames: string[]`(OR)
  - `keyword: string`
  - `sort: 'latest' | 'oldest' | 'amountDesc' | 'amountAsc'` — 정렬 기준(인라인 셀렉트, 즉시 반영). 금액순은 평면 목록
- 인라인 스테퍼는 `period`(month 모드)를, 정렬 셀렉트는 `sort`를 직접 갱신하여 시트 없이 즉시 반영. 시트의 '기간' 섹션도 같은 `period`를 편집
- 바텀시트는 `initial`에서 복제한 **draft** 상태를 편집하고 `[적용]` 시에만 페이지로 커밋(키 입력마다 재조회/재필터하지 않음)
- 조회 범위는 `resolvePeriod(period)`로 `{ start, end, label }` 도출 → `useCashbookEntriesInRange(coupleId, start, end)`
- 타입·카테고리·키워드는 클라이언트 사이드 필터링(`useFilteredEntries` 훅)
- URL 동기화·`localStorage` 기억은 없음(회고 흐름에서 매번 다른 조건을 보고 싶다는 가정)

## 기술 명세

### 데이터 페칭

```ts
// 선택된 기간(resolvePeriod)의 모든 항목
const { start, end } = resolvePeriod(filter.period); // 프리셋/커스텀 → Date 범위

query(
  collection(db, `couples/${coupleId}/cashbookEntries`),
  where('date', '>=', Timestamp.fromDate(start)),
  where('date', '<=', Timestamp.fromDate(end)),
  orderBy('date', 'desc')
);
```

> 키워드·OR 카테고리는 Firestore에서 부분일치/복수 동등을 단일 쿼리로 표현할 수 없으므로 범위 내 전량을 받아 클라이언트에서 필터링한다. 커플 단위 데이터량이 작아 허용하는 예외이며, 데이터가 커지면 cap/더보기를 도입한다.

### 월별 요약 계산 (클라이언트)

```ts
const totalIncome = entries
  .filter((e) => e.type === 'income')
  .reduce((sum, e) => sum + e.amount, 0);
const totalExpense = entries
  .filter((e) => e.type === 'expense')
  .reduce((sum, e) => sum + e.amount, 0);
const balance = totalIncome - totalExpense;
```

### 날짜별 그룹핑

```ts
// dayjs로 날짜별로 entries를 그룹핑 (dayjs(a).isSame(b, 'day'))
type GroupedEntries = {
  date: Date;
  entries: CashbookEntry[];
}[];
```

### 기간 필터

- 필터바에 **인라인 월 이동 스테퍼**(`‹ 2024년 1월 ›`) — month 모드일 때 노출, 이번 달에서 `›` 비활성(미래 불가)
- 범위 모드(최근 3개월/올해/직접 선택)에서는 화살표 없이 라벨만 표시하고, 라벨 탭 시 필터 시트가 열림
- 기본값: `이번 달`(`createDefaultFilterState()` — month 모드, 현재 연·월)
- `resolvePeriod(selection, now)`(`utils/date.ts`)가 `period`를 `{ start, end, label }`로 해석. `now`/`range`는 `useMemo`로 고정해 queryKey 안정성 확보
- custom 시작/종료는 미래 날짜도 허용(EntryForm이 미래 날짜 입력을 허용하므로 일관성 유지)

### CRUD 작업

| 작업 | Firebase 메서드               | TanStack Query        |
| ---- | ----------------------------- | --------------------- |
| 조회 | `getDocs` (또는 `onSnapshot`) | `useQuery`            |
| 추가 | `addDoc`                      | mutation + invalidate |
| 수정 | `updateDoc`                   | mutation + invalidate |
| 삭제 | `deleteDoc`                   | mutation + invalidate |

> `onSnapshot`을 사용하면 상대방이 추가한 내역이 실시간으로 반영됩니다.
> MVP에서는 `getDocs` + invalidate 방식도 허용.

### 카테고리 칩 UX

- 카테고리 프리셋은 `03-domain-models.md`의 `EXPENSE_CATEGORIES` / `INCOME_CATEGORIES` 참고
