# 사진 갤러리 2단계: 사진 업로드 + 상세 + 편집/삭제

> 상위 명세: `03-photo-gallery.md`
> 선행 단계: `03a-gallery-scaffold.md`

## 범위

- 사진 업로드 (폴더·태그 지정, 이미지 리사이즈, 진행 상태)
- 사진 상세 페이지 (`/photos/[id]`)
- 사진 편집 (캡션, 태그, 폴더 변경)
- 사진 삭제

## UI 구성

### 사진 업로드 — Bottom Sheet

갤러리 메인 헤더의 [+] 버튼 클릭 시 하단에서 Bottom Sheet 등장.

```
┌─────────────────────────┐
│  사진 추가               │
│                         │
│  ┌───────────────────┐  │
│  │                   │  │  ← 이미지 미리보기 영역
│  │  [사진 선택하기]   │  │  (파일 선택 전) / 선택 후 미리보기
│  └───────────────────┘  │
│                         │
│  폴더 *                  │
│  ┌───────────────────┐  │
│  │  제주도 여행    ▼  │  │  ← Select (폴더 목록)
│  └───────────────────┘  │
│                         │
│  촬영일                  │
│  ┌───────────────────┐  │
│  │  2024-01-15       │  │  ← date input (기본값: 오늘)
│  └───────────────────┘  │
│                         │
│  태그                    │
│  ┌───────────────────┐  │
│  │  #벚꽃 #여행  +   │  │  ← 자유 입력 + 기존 태그 자동완성
│  └───────────────────┘  │
│                         │
│  캡션 (선택)             │
│  ┌───────────────────┐  │
│  │                   │  │  ← textarea, placeholder: "이 순간을 기록해요"
│  └───────────────────┘  │
│                         │
│  ┌───────────────────┐  │
│  │  업로드            │  │  ← primary, 이미지+폴더 선택 시 활성화
│  └───────────────────┘  │
│                         │
└─────────────────────────┘
```

- **폴더 선택 필수**: 폴더가 없으면 먼저 폴더 생성 유도
- **태그 입력**: 텍스트 입력 후 Enter/쉼표로 태그 추가. 기존 태그 자동완성 드롭다운 제안
- 업로드 진행 시 progress bar 표시 (0% ~ 100%)
- 완료 시 Bottom Sheet 닫힘 + 갤러리 목록 자동 갱신

### 사진 상세 (/photos/[id])

```
┌─────────────────────────┐
│  ←                  [⋮] │  ← 뒤로가기 + 더보기 메뉴 (삭제, 편집)
├─────────────────────────┤
│                         │
│  ┌───────────────────┐  │
│  │                   │  │
│  │   (사진 원본)      │  │  ← 전체 너비, object-contain
│  │                   │  │
│  └───────────────────┘  │
│                         │
│  📁 제주도 여행          │  ← 폴더명 (클릭 → 폴더 상세 이동)
│  #벚꽃 #바다             │  ← 태그 (클릭 → 태그 상세 이동)
│                         │
│  2024년 1월 15일         │  ← takenAt 포맷
│  홍길동이 올림           │  ← uploadedBy 표시 (Avatar + displayName)
│                         │
│  이 날 정말 행복했어 ☺   │  ← caption (없으면 영역 숨김)
│                         │
└─────────────────────────┘
```

- 더보기 메뉴 [⋮]:
  - **편집**: 캡션, 태그, 폴더 변경 (Bottom Sheet)
  - **삭제**: 확인 모달 후 삭제
- 폴더명 클릭 → `/photos/folder/[folderId]`
- 태그 클릭 → `/photos/tag/[tagName]`

```
│  │  저장              │  │  ← primary
│  └───────────────────┘  │
│                         │
└─────────────────────────┘
```

## 기술 명세

### 이미지 업로드 흐름

1. 사용자가 파일 선택 (`<input type="file" accept="image/*">`)
2. 파일 크기가 **1.5MB 초과**인 경우에만 클라이언트에서 이미지 리사이즈 (최대 1920px, quality 85%)
   - 1.5MB 이하는 원본 그대로 업로드
   - `browser-image-compression` 라이브러리 사용 (`pnpm --filter web add browser-image-compression`)
3. Firebase Storage에 업로드 → `storageUrl` 획득
4. Firestore에 Photo 문서 생성
   - `folderId`: 선택된 폴더 ID
   - `tags`: 입력된 태그 배열
   - `width`, `height`는 업로드 전 이미지에서 추출

```ts
// Storage 업로드 경로
`couples/${coupleId}/photos/${photoId}/original.${ext}`;
```

### 태그 자동완성

```ts
// 커플의 모든 사진에서 사용된 태그 목록 조회
// 클라이언트 캐싱: TanStack Query로 태그 목록 캐시
// 사진 업로드/편집 시 invalidate
```

> 태그 목록은 별도 컬렉션 없이 사진의 `tags` 배열에서 클라이언트 사이드로 집계한다.

### 업로드 진행 상태

- Bottom Sheet 내 progress bar 표시 (`0% ~ 100%`)
- 완료 시 Bottom Sheet 닫힘, 갤러리 목록 자동 갱신 (TanStack Query invalidate)

### 사진 삭제

- 상세 페이지 우상단 `[⋮]` 메뉴에서 "삭제" 선택
- 확인 모달 표시: "이 사진을 삭제하면 복구할 수 없어요. 삭제할까요?"
- 확인 시:
  1. Storage 파일 삭제
  2. Firestore 문서 삭제
  3. `/photos`로 이동

### 사진 편집

- 상세 페이지 우상단 `[⋮]` 메뉴에서 "편집" 선택
- Bottom Sheet로 캡션, 태그, 폴더 변경 가능
- 폴더 변경 = 사진을 다른 폴더로 이동

### 이미지 표시 최적화

- 상세 페이지: `sizes="100vw"`
