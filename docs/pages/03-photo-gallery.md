# 페이지 명세: 사진 갤러리 (종합)

## 목적

커플이 함께 찍은 사진을 **폴더**로 분류하고, **태그**로 횡단 검색하며, 슬라이드쇼로 감상하는 공간.

---

## 라우트 구조

| 경로 | 설명 |
| --- | --- |
| `/photos` | 갤러리 메인 (탭: 전체 / 폴더 / 태그) |
| `/photos/folder/[folderId]` | 특정 폴더의 사진 목록 |
| `/photos/tag/[tagName]` | 특정 태그의 사진 목록 (폴더 무관) |
| `/photos/[id]` | 사진 상세 |
| `/photos/slideshow?source=folder&id=[folderId]` | 슬라이드쇼 (폴더 기준) |
| `/photos/slideshow?source=tag&id=[tagName]` | 슬라이드쇼 (태그 기준) |

---

## 사용자 스토리

- 커플로서, 우리가 함께한 사진들을 **폴더별로 정리**해서 보고 싶다.
- 커플로서, 여행 > 일본 > 도쿄처럼 **폴더 안에 폴더**를 만들어 더 세밀하게 정리하고 싶다 (최대 5단계).
- 커플로서, 사진을 올릴 때 **내가 올린 사진**인지 **연인이 올린 사진**인지 구분하고 싶다.
- 커플로서, 여러 폴더에 흩어진 사진을 **태그 기준으로 모아**보고 싶다.
- 커플로서, 폴더나 태그 단위로 사진을 **슬라이드쇼**로 감상하고 싶다.
- 커플로서, 새 사진을 쉽게 업로드하고 캡션·태그를 달고 싶다.

## 폴더 중첩 정책

- 루트 폴더(`parentFolderId = null`)와 하위 폴더를 구분한다. 하위는 임의의 폴더 아래에 만들 수 있고, 최대 깊이는 5단계(`depth 0~4`).
- 탐색은 **Drill-down** 방식: 폴더 상세 페이지 상단에 하위 폴더 카드 목록, 아래에 현재 폴더 사진 그리드.
- 폴더 상세 상단에 **Breadcrumb**으로 현재 위치를 표시 (루트 → 조상 → 현재). 루트/조상은 링크, 현재는 비링크.
- 사진은 정확히 한 폴더에만 속한다. 부모 폴더를 열어도 하위 폴더의 사진은 보이지 않는다(폴더 단위 격리).
- 폴더 삭제는 **재귀 삭제**: 하위 폴더와 그 안의 사진(Storage 파일 포함)을 모두 제거한다. 확인 모달에 "하위 폴더 N개, 사진 M장"을 명시.
- 폴더 이동(parent 변경)은 이번 범위 밖. 별도 기능으로 분리.

## 마이그레이션 (기존 데이터)

최초 배포 시 1회 실행: `pnpm --filter web exec tsx scripts/migrate-folders-nested.ts`

기존 모든 폴더 문서에 `parentFolderId: null`, `depth: 0`, `path: []`를 추가한다.
