# 페이지 명세: 온보딩 (/onboarding)

## 목적

최초 로그인 후 커플 연결이 완료되지 않은 사용자를 안내하는 페이지.
이 페이지를 통과해야만 서비스 본 기능에 접근할 수 있습니다.

---

## 사용자 스토리

- 신규 사용자로서, 상대방과 연결할 수 있는 쉬운 방법을 원한다.
- 초대 코드를 받은 사용자로서, 코드를 입력해서 상대방이 만든 공간에 합류하고 싶다.

---

## UI 구성

### 단계 1 — 선택 화면

```
┌─────────────────────────┐
│                         │
│   UANDI                 │  ← 로고
│   우리 둘만의 공간       │
│                         │
│   ┌───────────────────┐ │
│   │  새 커플 공간 만들기│ │  ← primary 버튼
│   └───────────────────┘ │
│                         │
│   ┌───────────────────┐ │
│   │  초대 코드 입력하기 │ │  ← secondary 버튼
│   └───────────────────┘ │
│                         │
└─────────────────────────┘
```

### 단계 2A — 새 커플 공간 만들기

"새 커플 공간 만들기" 클릭 → 로딩 후 초대 코드 표시

```
┌─────────────────────────┐
│  ← 뒤로                 │
│                         │
│  초대 코드               │
│                         │
│  ┌───────────────────┐  │
│  │    A B 1 2 C D    │  │  ← 코드 표시 (큰 폰트, letter-spacing)
│  └───────────────────┘  │
│  [복사하기] 버튼         │
│                         │
│  상대방에게 이 코드를    │
│  공유해 주세요.          │
│  코드는 48시간 후 만료됩니다 │
│                         │
│  ┌───────────────────┐  │
│  │  상대방이 입력했어요│  │  ← secondary, 대기 중 상태
│  └───────────────────┘  │
│                         │
└─────────────────────────┘
```

> "상대방이 입력했어요" 버튼은 실제로 버튼이 아니라 안내 문구입니다.
> 상대방이 코드를 입력하면 Firestore 실시간 리스너로 자동 감지되어 대시보드로 이동합니다.

## 기술 명세

### 상태 관리

```ts
type OnboardingStep =
  | 'select' // 초기 선택 화면
  | 'create_pending' // 코드 생성 완료, 상대방 대기 중
  | 'join'; // 코드 입력 화면
```

### 동작 흐름

**새 커플 공간 만들기**

1. `createCouple(uid)` 호출
2. 반환된 `coupleId`, `inviteCode`를 화면에 표시
3. `couples/{coupleId}` 문서를 Firestore 실시간 리스너로 구독
4. `memberUids.length === 2`가 되면 → `user.coupleId` 업데이트 확인 후 `/`으로 이동

**초대 코드 입력**

1. 6자리 입력 완료 시 "연결하기" 버튼 활성화
2. `joinCoupleByInviteCode(uid, code)` 호출
3. 성공 → `/`으로 이동
4. 실패 → 에러 유형별 메시지 표시

### 에러 메시지

| 에러           | 표시 메시지                                                  |
| -------------- | ------------------------------------------------------------ |
| 코드 없음      | "존재하지 않는 초대 코드예요"                                |
| 코드 만료      | "초대 코드가 만료됐어요. 상대방에게 새 코드를 요청해 주세요" |
| 이미 2명       | "이미 연결된 커플이에요"                                     |
| 자기 자신 코드 | "내가 만든 코드예요. 상대방에게 공유해 주세요"               |

### OTP 입력 컴포넌트

shadcn의 `input-otp` 컴포넌트를 사용합니다.

```bash
# packages/ui에서 실행
pnpm dlx shadcn@latest add input-otp
```

```tsx
import { InputOTP, InputOTPGroup, InputOTPSlot } from '@uandi/ui';

<InputOTP maxLength={6}>
  <InputOTPGroup>
    <InputOTPSlot index={0} />
    <InputOTPSlot index={1} />
    <InputOTPSlot index={2} />
    <InputOTPSlot index={3} />
    <InputOTPSlot index={4} />
    <InputOTPSlot index={5} />
  </InputOTPGroup>
</InputOTP>
```

- 한 칸 입력 시 자동으로 다음 칸으로 포커스 이동 (shadcn이 처리)
- Backspace 입력 시 이전 칸으로 포커스 이동 (shadcn이 처리)
- 붙여넣기(paste) 시 6자리를 각 칸에 분배 (shadcn이 처리)
