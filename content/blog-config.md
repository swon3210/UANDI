# 블로그 글 작성 설정

이 파일은 `/generate-devlog` 스킬이 글을 생성할 때 참고하는 설정입니다.
내용을 수정하면 다음 글 생성부터 반영됩니다.

---

## 대상 독자

- 주니어~미드레벨 개발자
- AI를 활용한 서비스 개발에 관심 있는 사람
- 기술적 깊이보다는 실제 경험과 과정에 관심 있는 사람

---

## 톤 & 어조

- 개발자가 동료에게 이야기하듯 **자연스러운 한글**
- 격식 없이 편하게, 하지만 내용은 정확하게
- 삽질이나 실수도 솔직하게 공유 — 이것이 개발 일지의 가치
- 과도한 전문 용어는 피하되, 필요한 기술 용어는 그대로 사용

---

## 글 길이

- **목표**: 1000~2000자 (읽는 데 3~5분)
- **최소**: 800자
- **최대**: 3000자

---

## 섹션 구조

### 필수 섹션

| 섹션 | 설명 |
|------|------|
| **배경** | 이 작업을 하게 된 이유, 해결하려는 문제. 독자가 맥락을 이해할 수 있도록. |
| **구현 내용** | 무엇을 만들었는지, 주요 변경사항. 핵심 코드 블록 포함 (1~3개). |
| **배운 것** | 과정에서 새로 알게 된 점, 삽질 경험, 다음에 다르게 할 점. |

### 선택 섹션 (필요할 때만 포함)

| 섹션 | 언제 포함하는가 |
|------|----------------|
| **기술적 결정** | 대안이 있었고 특정 선택을 한 이유가 있을 때 |
| **트러블슈팅** | 예상치 못한 버그나 어려움을 해결한 과정이 있을 때 |
| **다음 단계** | 후속 작업이 명확하고, 독자에게 시리즈 느낌을 줄 때 |
| **참고 자료** | 도움이 된 문서, 블로그, 라이브러리 링크가 있을 때 |

---

## 코드 블록 규칙

- **최대 개수**: 글 하나에 3개 이하
- **최대 줄 수**: 코드 블록 하나당 20줄 이하
- 전체 파일 붙여넣기 금지 — 핵심 부분만 발췌
- 반드시 언어 태그 포함 (```typescript, ```bash 등)
- 코드 앞뒤에 설명 추가 (코드만 덩그러니 두지 않기)

---

## 태그 규칙

- 프로젝트 기능 영역: `photos`, `cashbook`, `auth`, `blog`, `dashboard`, `onboarding`
- 기술 키워드: `performance`, `testing`, `infrastructure`, `design-system`, `firebase`
- 글당 2~4개 태그 사용
