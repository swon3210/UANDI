# 블로그 글 작성 설정

이 파일은 `/generate-devlog` 스킬이 글을 생성할 때 참고하는 설정입니다.
내용을 수정하면 다음 글 생성부터 반영됩니다.

---

## 대상 독자

- 주니어~미드레벨 개발자
- AI를 활용한 서비스 개발에 관심 있는 사람
- 기술적 깊이보다는 실제 경험과 과정에 관심 있는 사람

---

## 목소리 & 어조 (이 블로그에서 제일 중요)

레퍼런스: **이동욱(jojoldu)의 담백한 1인칭 회고체** (예: https://jojoldu.tistory.com/775).
"딱 이 스타일"로 합의됨. 아래 원칙을 frontmatter·길이·섹션 구조보다 **우선**한다.

1. **1인칭으로 쓴다.** 화자는 "나". 공동작업자(아내 등)는 "우리"가 아니라 등장인물로 둔다. "우리"라는 추상명사 뒤에 숨지 않는다.
2. **구체적 장면으로 연다.** 추상적 요약("돈이 안 모인다") 대신 실제 사건 하나로 시작한다("신혼여행을 다녀오니 통장이 0이었다"). 도입부 첫 두세 문장에서 장면이 보여야 한다.
3. **글마다 척추(thesis) 하나.** 모든 단락이 한 줄짜리 깨달음으로 수렴하게 한다(예: "빠름이 아니라 정확함이 먼저였다"). 백화점식 나열 금지.
4. **솔직하게, 부끄러운 것까지.** 삽질·실패·내 가정이 틀렸던 순간을 그대로 쓴다. 미구현·미완성 기능은 "아직 못 했다"고 적는다. 잘된 것만 쓰면 광고가 된다.
5. **자조는 농담이 아니라 솔직함+구체성에서 나온다.** 억지 드립·이모지·과장된 감탄 금지. 내 멍청했던 선택을 정확히 묘사하면 유머는 알아서 따라온다.
6. **꾸밈없는 평서문.** 문장을 짧게, 리듬에 강약을 둔다. 과도한 전문용어는 피하되 필요한 기술 용어는 그대로 쓴다.

> 핵심 한 줄: **솔직한 1인칭 + 구체적 장면 + 척추 하나.** 셋이 갖춰지면 개성은 저절로 생긴다.

## 섹션 구조

### 필수 섹션

| 섹션 | 설명 |
|------|------|
| **배경** | 이 작업을 하게 된 이유, 해결하려는 문제. 독자가 맥락을 이해할 수 있도록. |
| **구현 내용** | 무엇을 만들었는지, 주요 변경사항. 핵심 코드 블록 포함 (1~3개). |
| **배운 것** | 과정에서 새로 알게 된 점, 삽질 경험, 다음에 다르게 할 점. |

### 선택 섹션 (필요할 때만 포함)

| 섹션 | 언제 포함하는가 |
|------|----------------|
| **기술적 결정** | 대안이 있었고 특정 선택을 한 이유가 있을 때 |
| **트러블슈팅** | 예상치 못한 버그나 어려움을 해결한 과정이 있을 때 |
| **다음 단계** | 후속 작업이 명확하고, 독자에게 시리즈 느낌을 줄 때 |
| **참고 자료** | 도움이 된 문서, 블로그, 라이브러리 링크가 있을 때 |

## 태그 규칙

- 프로젝트 기능 영역: `photos`, `cashbook`, `auth`, `blog`, `dashboard`, `onboarding`
- 기술 키워드: `performance`, `testing`, `infrastructure`, `design-system`, `firebase`
- 한글 태그도 허용 (예: `가계부`, `회고`, `AI`) — 기존 글 관례
- 글당 2~4개 태그 사용

## Frontmatter (필수 필드)

```yaml
---
title: '<한글 제목>'
date: 'YYYY-MM-DD'
summary: '<1~2문장 한글 요약 — 글의 척추가 드러나게>'
category: '<아래 5개 중 하나>' # 누락/오타 시 빌드 실패
tags: ['<2~4개>']
draft: true
---
```

- `category`는 반드시 다음 중 하나 (파서가 검증, 틀리면 빌드가 throw):
  - `tech-how` — 기술 구현 노트
  - `product-how` — 프로덕트 맥락 안의 기술
  - `tech-decision` — 왜 이 스택을 골랐는가
  - `product-decision` — 사용자/실사용에서 배운 것
  - `essay` — 개발자로서의 생각
- 선택 필드: `featured`(홈 대표글), `series` + `seriesOrder`(등록된 시리즈 키에 속할 때만)
